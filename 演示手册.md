## 三、患者端 H5 演示路径

> 入口：`http://localhost:8000/login` → 患者账号登录 → 自动跳转 `/h5/home`

### 3.1 个人档案与慢病管理 `/h5/profile`

**操作步骤：**
1. 页面自动加载已有健康档案（身高 / 体重 / 生活方式等）
2. 点击「编辑档案」→ 填写 / 修改基础信息 → 点击「保存」
3. 点击「添加慢病」→ 选择病种（高血压 / 2 型糖尿病）→ 填写确诊时间、用药记录 → 保存

---

### 3.2 指标录入与历史趋势

| 页面 | 路径 |
|---|---|
| 历史趋势图 | `/h5/indicators` |
| 录入新指标 | `/h5/indicators/add` |

**操作步骤：**
1. 历史页切换指标类型（血压 / 血糖 / 体重 / 腰围），图表联动更新
2. 点击右上角「+」进入录入页
3. 选择指标类型 → 填写数值（血压需填收缩压 + 舒张压）→ 选场景 → 提交
4. **⚡ 危险值自动触发预警**（收缩压 ≥ 140 / 血糖 ≥ 7.0），管理端可见

**预警阈值速查：**

| 指标 | 低危 | 中危 | 高危 |
|---|---|---|---|
| 收缩压 (mmHg) | ≥ 140 | ≥ 160 | ≥ 180 |
| 血糖 (mmol/L) | ≥ 7.0 | ≥ 8.0 | ≥ 11.1 |

---

### 3.3 中医体质辨识

| 页面 | 路径 |
|---|---|
| 评估入口 / 最新结果 | `/h5/constitution` |
| 作答问卷 | `/h5/constitution/assess` |
| 查看报告 | `/h5/constitution/result/{id}` |

**操作步骤：**
1. 首次进入显示「开始评估」按钮
2. 进入答题页 → 分页展示（每页 10 题，共 72 题）→ 支持断点续答
3. 全部作答后点击「提交评估」→ 系统自动评分
4. 跳转结果页，展示：
   - 主导体质（如：气虚质）、兼夹体质
   - 各体质转化分
   - **自动生成个性化调护建议方案**

**九种体质：**
`平和质` `气虚质` `阳虚质` `阴虚质` `痰湿质` `湿热质` `血瘀质` `气郁质` `特禀质`

---

### 3.4 调护建议 `/h5/recommendation`

**操作步骤：**
1. 自动读取最新活跃方案（体质辨识提交后自动生成）
2. 按五大类展示（点击展开 / 折叠）：

| 类别 | 说明 |
|---|---|
| 起居作息 | 睡眠、作息节律建议 |
| 饮食调护 | 宜食 / 忌食食物 |
| 运动导引 | 适宜运动方式与强度 |
| 情志调护 | 心理调节方法 |
| 外治非药 | 穴位保健、理疗方法 |

---

### 3.5 随访打卡 `/h5/followup`

**操作步骤：**
1. 首次进入 → 点击「开启随访计划」→ 选择病种 → 系统自动生成 30 天计划
2. 今日任务卡片展示各类任务（指标上报 / 运动 / 用药 / 饮食 / 睡眠）
3. 点击任务「打卡」→ 填写打卡值（如步数）→ 提交
4. 顶部进度条实时显示当日完成比例
5. 切换「计划列表」查看历史计划与依从率

> **后台自动标记**：每天凌晨 1 点将昨日未打卡任务自动标记为「漏打」

---

### 3.6 科普内容

| 页面 | 路径 |
|---|---|
| 内容列表 | `/h5/content` |

**操作步骤：**

2. 点击标签（体质 / 饮食 / 运动等）筛选分类

## 四、管理端 PC 演示路径

> 入口：`http://localhost:8000/login` → 管理员账号登录 → 自动跳转管理端

### 4.1 统计报表 `/gui/admin/stats`（首选演示起点）

**展示内容：**
- KPI 卡片：患者总数 / 在管患者 / 开放预警数 / 整体依从率
- 体质分布饼图（九体质占比）
- 预警严重程度分布（高危 / 中危 / 低危进度条）

---

### 4.2 预警处置台

| 页面 | 路径 |
|---|---|
| 预警列表 | `/gui/admin/alerts` |
| 预警详情 & 处置 | `/gui/admin/alerts/{id}` |

**操作步骤：**
1. 按严重程度（HIGH / MEDIUM / LOW）和状态（OPEN / ACKED / CLOSED）筛选
2. 点击「处置」进入详情页
3. 填写处置备注 → 点击「确认预警（ACK）」→ 再点「关闭预警」
4. **状态机**：`OPEN → ACKED → CLOSED`（不可逆）

---

### 4.3 患者管理 `/gui/admin/patients`

**操作步骤：**
1. 搜索框输入姓名 / 手机号实时查询
2. 状态下拉筛选（全部 / 启用 / 禁用）
3. 点击「禁用」→ 确认弹窗 → 患者立即无法登录
4. 点击「启用」恢复访问权限

---

### 4.4 随访质控台 `/gui/admin/followup`

**展示内容：**
- 顶部概览：在管患者数 / 整体依从率 / 开放预警
- 全患者随访依从性列表，**按依从率升序**（最差排最前，便于重点关注）
- 每行显示：患者姓名 / 手机、病种、计划周期、完成 / 总数、漏打次数、依从率进度条

**依从率色标：**
- 🟢 绿色 ≥ 80%
- 🟡 黄色 60% ~ 80%
- 🔴 红色 < 60%

---

### 4.5 内容管理

| 页面 | 路径 |
|---|---|
| 内容列表 | `/gui/admin/content` |
| 新建文章 | `/gui/admin/content/new` |
| 编辑文章 | `/gui/admin/content/{id}/edit` |

**操作步骤：**
1. 按状态筛选（草稿 / 待审核 / 已发布 / 下线）
2. 点击「新建」→ 填写标题 / 摘要 / 正文 / 标签 → 「保存草稿」
3. 「提交审核」→ 状态变为待审核
4. 「发布」→ 患者端立即可见
5. 「下线」→ 患者端隐藏
6. **状态机**：`草稿 → 待审核 → 已发布 → 下线`

---

### 4.6 审计日志 `/gui/admin/audit`

**操作步骤：**

2. 查看每条操作：操作者、IP、操作前后值
3. 支持分页翻页

**已覆盖审计操作类型（17 类）：**
`LOGIN` `REGISTER` `CONSENT` `UPDATE_PROFILE` `ADD_DISEASE` `ADD_INDICATOR`
`START_ASSESSMENT` `SUBMIT_ASSESSMENT` `START_FOLLOWUP_PLAN` `CHECKIN`
`ACK_ALERT` `CLOSE_ALERT` `CREATE_CONTENT` `SUBMIT_REVIEW`
`PUBLISH_CONTENT` `OFFLINE_CONTENT` `TOGGLE_USER_ACTIVE`

## 八、环境切换说明

### 当前模式（SQLite 演示）
```
DATABASE_URL=sqlite+aiosqlite:///./demo.db
DEMO_MODE=true
```

### 切换 PostgreSQL 生产模式
```bash
# 1. 修改 .env
DATABASE_URL=postgresql+asyncpg://user:pass@localhost:5432/tcm
DEMO_MODE=false

# 2. 运行迁移
py -m alembic upgrade head

# 3. 手动 Seed（可选）
py -X utf8 scripts/seed_demo.py
```

### 运行测试
```bash
# 全量回归（使用 SQLite in-memory，无需数据库）
py -m pytest tests/ -v

# 只跑单元测试
py -m pytest tests/services/ -v

# 只跑集成测试
py -m pytest tests/integration/ -v
```
