## 三、核心演示：三条商业价值链（约 15 分钟）

> 这是系统最核心的竞争力，建议从这里开始演示。

### 🔗 价值链 1：诊中 AI 识别 → 一键下达方案

**场景**：医生正在接诊，系统自动扫描患者数据并生成调理方案。

#### 方式A：综合就诊台（推荐演示入口）

> 路径：`/gui/admin/visit-station`（侧边栏 → 中医管理 → 综合就诊台）

1. **搜索患者**：在左列搜索框输入姓名（如"张"），点击患者卡片选中
2. **查看临床数据**：中左列自动加载该患者的检验报告 / 就诊记录 / 处方历史
3. **触发 AI 分析**：点击中列「开始 AI 分析」→ 系统调用 Claude 或规则引擎
4. **查看风险结果**：
   - 风险等级徽章（高危红 / 中危橙 / 低危绿）
   - 风险因素列表（如"空腹血糖偏高 + 痰湿体质"）
   - AI 综合摘要
5. **生成调理方案**：点击右列「生成中医方案」→ AI 输出 Markdown 格式方案（含饮食/运动/中药/穴位）
6. **一键下达**：填写方案标题 → 点击「下达给患者」→ 系统同步：
   - 创建指导记录（`guidance_records`）
   - 推送通知到患者手机端（`PLAN_ISSUED`）
   - 自动创建随访任务（可选）

#### 方式B：AI 风险诊断（独立精细操作）

> 路径：`/gui/admin/risk/plan`（侧边栏 → 中医管理 → AI风险诊断）

- 与就诊台流程相同，但为独立页面，适合非诊中场景的主动复查

---

### 🔗 价值链 2：患者接收推送 → 查看方案

**场景**：患者回家后收到方案通知，查看并与医生咨询。

> 切换账号：退出 → 患者账号（patient@tcm）登录 → `/h5/home`

1. **首页铃铛**：欢迎卡片右上角出现🔔红点（未读通知数）
2. **通知中心**：点击铃铛 → `/h5/notifications`
   - 列表展示各类通知：🚨风险预警 / 📋方案下达 / 📅随访提醒 / 💬咨询回复
   - 点击"方案下达"通知 → 跳转调理方案详情页
3. **方案详情** `/h5/plan/{id}`：
   - 医生生成的 Markdown 方案渲染展示（饮食/运动/中药/穴位）
   - 底部「就方案咨询医生」按钮 → 进入咨询发起页

---

### 🔗 价值链 3：患者在线咨询 → 医生回复 → 复诊转化

**场景**：患者有疑问，发起在线咨询；医生利用碎片时间回复（可收费）。

**患者侧**（patient@tcm）：

1. 首页快捷入口「在线咨询 💬」→ `/h5/consultation/new`
2. 填写标题 + 问题描述 + 选择普通/紧急 → 提交
3. 跳转咨询详情页 `/h5/consultation/{id}` → 可继续追问

**医生侧**（切换回 admin@tcm）：

1. 工作台底部「待回复咨询」列表 → 点击进入
2. 或：侧边栏 → 在线服务 → 在线咨询 `/gui/admin/consultations`
3. 咨询列表展示：状态/优先级徽章 + 末条消息预览
4. 点击进入咨询详情 `/gui/admin/consultations/{id}`：
   - 气泡对话视图（患者消息左灰/医生消息右蓝）
   - 输入框回复（Ctrl+Enter 快捷发送）
5. **回复后**：系统自动推送「💬 咨询回复」通知到患者手机端

## 四、Chrome 插件演示（诊中嵌入 HIS）

> 这是系统与传统方案的核心差异化：**医生无需切换界面**。
> 插件当前版本：**v2.0**（三 Tab 布局 + 证据溯源 + 状态机管理）

### 4.1 安装插件

1. Chrome 访问 `chrome://extensions/`
2. 右上角开启「开发者模式」
3. 点击「加载已解压的扩展程序」→ 选择项目目录下的 `chrome_extension/` 文件夹
4. 工具栏出现绿色 🌿 图标，安装完成
5. 支持 Chrome 93+ 及基于 Chromium 的浏览器（Edge、Brave 等）

### 4.2 首次配置

1. 点击工具栏插件图标，弹出配置页
2. **后端服务地址**：填写 `http://localhost:8010`（默认值为 8001，需改为 8010）
3. **HIS患者ID参数名**：默认 `patient_id,pid,id,patientId,patient`，通常无需修改
4. 点击「保存配置」

### 4.3 使用前登录

浏览器访问 `http://localhost:8010/login` 用 admin@tcm 登录（插件通过 Cookie 认证）。

### 4.4 演示方法（无需真实 HIS）

**方法一：URL 参数自动触发（推荐）**

在地址栏访问任意带 `patient_id` 参数的页面：
```
http://localhost:8010/gui/admin/archives?patient_id=张
```
→ 插件检测到参数后自动向后台发起患者搜索 + AI 风险分析，右侧侧边栏展开显示结果

**方法二：弹窗手动搜索**

1. 打开任意页面（如 `http://localhost:8010/gui/admin/archives`）
2. 点击工具栏插件图标 → 在「手动搜索患者」框输入患者姓名（如"李"）
3. 点击「搜索」→ 侧边栏自动展开并显示分析结果

> 防抖：URL 频繁变化时插件有 600ms 防抖，避免重复请求。

### 4.5 侧边栏三 Tab 详解

插件侧边栏固定在页面右侧，点击 ◀ 可收起为绿色竖条，点击竖条再展开。

#### Tab 1：当前风险

| 区域 | 内容说明 |
|---|---|
| 患者信息卡 | 姓名 + 档案编号 + 风险等级徽章（高🔴/中🟠/低🟢） |
| 风险证据 | 每条证据可点击展开溯源（检测值/参考范围/来源/日期） |
| AI 摘要 | 风险分析自然语言摘要（前 150 字） |
| 下达按钮 | 「⚡ 确认下达方案」→ 弹出确认框 |
| 跳转按钮 | 「完整分析 ↗」→ 新标签页打开平台完整分析页 |

**下达方案确认框**：
- 显示患者姓名 + 方案内容预览
- 勾选「自动创建随访提醒」（默认开启）+ 选择随访时间（3/7/14/30天）
- 确认后系统创建指导记录并推送患者通知；若勾选随访，则自动创建随访任务

#### Tab 2：历史方案

展示该患者所有历史下达记录，支持**方案状态流转**：

| 当前状态 | 可操作 |
|---|---|
| 已下达（ISSUED） | → 标记进行中 |
| 进行中（IN_PROGRESS） | → 标记已随访 / 调整方案 |
| 已随访（FOLLOWED_UP） | → 标记已复评 / 完结 |
| 已复评（RE_ASSESSED） | → 完结 |
| 已完结（COMPLETED） | — |

每条记录显示：状态图标 + 下达日期 + 方案摘要 + 随访日期（如有）

#### Tab 3：数据指标

| 指标 | 说明 |
|---|---|
| 总分析次数 | 本机构累计 AI 分析触发次数 |
| 已下达方案 | 通过插件下达的方案总数 |
| 方案下达率 | 中高风险患者中已下达方案的比例（进度条） |
| 随访完成率 | 已创建随访任务中已完成比例（进度条） |

### 4.6 插件调用的 API 接口

| 接口 | 说明 |
|---|---|
| `GET /tools/archive/archives?q={keyword}&page_size=1` | 搜索患者档案 |
| `POST /tools/risk/analyze/{archive_id}` | 触发 AI 风险分析（15s超时，超时后回退读历史结果） |
| `GET /tools/risk/result/{archive_id}` | 获取已有风险分析结果 |
| `POST /tools/risk/plan/issue` | 下达调理方案（含auto_followup_days参数） |
| `GET /tools/risk/plans/{archive_id}?page_size=10` | 获取历史方案列表 |
| `PATCH /tools/risk/plans/{record_id}/state` | 更新方案状态 |
| `GET /tools/risk/stats` | 获取业务统计数据 |

所有接口通过 `credentials: 'include'` 携带浏览器 Cookie 认证。

## 七、后台自动化任务

系统启动后在后台自动运行（无需手动触发）：

| 任务 | 频率 | 说明 |
|---|---|---|
| 扫描新检验报告 | 每 5 分钟 | 检测 HIS 新入 LAB_REPORT，自动触发 AI 风险分析 + 预警 |
| 漏打卡标记 | 每日凌晨 1:00 | 将昨日 PENDING 打卡标记为 MISSED |
| 随访逾期提醒 | 每日凌晨 2:00 | 有逾期打卡的患者推送「随访提醒」（7天内不重复） |
| 未复诊提醒 | 每日凌晨 3:00 | 收到方案后 90 天内无就诊记录的患者推送「建议复诊」 |

## 十一、AI 功能配置

### 有 ANTHROPIC_API_KEY（完整 AI 体验）

在 `.env` 中配置：
```
ANTHROPIC_API_KEY=sk-ant-xxx
```

- AI 风险分析：调用 Claude 读取检验报告 + 体质评估，生成自然语言风险分析
- AI 方案生成：Claude 输出个性化中医调理方案（饮食/运动/中药/穴位）
- Ctrl+K 智能助手：全局快捷键唤起，支持自然语言操作系统功能

### 无 API Key（规则引擎兜底）

- 风险分析：基于硬编码阈值规则（血糖≥7.0 / 血压≥140 等）
- 方案生成：返回体质对应的默认模板方案
- 功能可用，效果受限
