# 中医慢病与治未病平台：三端页面工程化清单

---

## 一、医生端插件页面清单（HIS 内主入口）

> 设计原则：
> 1. 插件只做诊中高价值动作
> 2. 主面板只展示最小决策集
> 3. 详细信息进入抽屉
> 4. AI 操作台常驻底部
> 5. 所有写操作必须确认

---

### 1. 插件主面板（首页）

**页面定位**  
医生在 HIS 接诊当前患者时的默认页面，用于快速完成判断、方案确认、分发、复评安排。

**核心模块**  
1. 关键提醒区  
2. 体质/风险结论区  
3. 方案状态区  
4. 复评与召回区  
5. 底部 AI 操作台

**核心字段/内容**

#### 1）关键提醒区
- alert_title
- severity
- reason_1line
- source_system
- source_date
- evidence_ref

#### 2）体质/风险结论区
- tcm_attributes（最多3个）
- tcm_strength
- risk_topics（最多2个）
- risk_level
- confirmed_status
- last_updated_at

#### 3）方案状态区
- plan_status
- plan_version
- plan_headlines（最多5条）
- preview_ready
- distribution_status

#### 4）复评与召回区
- next_review_date
- next_review_goal
- followup_status
- recall_status
- recall_policy_1line

#### 5）底部 AI 操作台
- text_input
- voice_input
- transcript_preview
- ai_result_cards

**主要按钮/动作**
- 查看证据
- 查看证据链
- 确认体质判断
- 确认风险判断
- 生成方案草稿
- 预览方案
- 确认方案
- 一键分发
- 查看分发结果
- 创建/调整随访
- 查看召回建议
- 接受/忽略召回建议
- 发送 AI 指令
- 语音录入

**关联抽屉**
- 临床摘要抽屉
- 证据链抽屉
- 补充采集抽屉
- 方案预览抽屉
- 分发结果抽屉
- 随访管理抽屉
- 召回建议抽屉
- 患者工作台抽屉

## 四、页面之间的主链路

### 医生端插件主链路
插件主面板  
→ 查看临床摘要/证据链  
→ 补充采集  
→ 生成体质/风险  
→ 方案预览  
→ 方案确认  
→ 分发结果  
→ 随访管理/召回建议

### 管理端主链路
首页总览  
→ 患者管理页  
→ 患者详情页  
→ 套餐/模板/规则配置  
→ 运营漏斗  
→ 审计日志/AI日志

### 患者端主链路
首页  
→ 当前方案  
→ 今日任务  
→ 打卡与反馈 / 量表  
→ 复评提醒  
→ 升级/召回建议  
→ 我的记录 / 消息
