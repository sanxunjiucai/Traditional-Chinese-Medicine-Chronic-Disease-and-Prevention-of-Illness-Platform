## 关键设计决策

1. **渐进式重构** - 保留 v1 系统，v2 独立演进
2. **表名隔离** - 使用 `_v2` 后缀避免冲突
3. **JSON 灵活性** - 关键字段用 JSON 存储便于扩展
4. **服务层分离** - 业务逻辑从 API 层抽离
5. **证据链优先** - 所有判断必须有证据支撑
