# 中医慢病与治未病平台（全量功能清单）

> 目标：覆盖“慢病管理 + 治未病（体质辨识/调护）+ 院后康复 + 医院运营与合规”全量场景，便于招标对标与后续迭代拆包。
>
> 说明：清单按模块分组；每个模块包含**用户端/专业端/管理端**能力，并标注关键状态机、常见集成点与高风险合规项。

---

## 0. 端与角色

### 0.1 系统端（载体）
- **用户端（移动端）**：小程序/H5/APP（患者、家属/照护者）
- **专业端（PC/Pad）**：医生/中医师/健康管理师/随访专员/康复师
- **运营管理端（PC）**：内容运营、模板配置、质控、数据分析、权限审计
- **机构对接端（接口/网关）**：HIS/EMR、LIS、PACS、体检系统、设备平台、医保/支付、消息平台

### 0.2 角色
- 患者/健康用户、家属/照护者（代填）
- 中医师（治未病/中医门诊）、全科/专科医生、护士
- 健康管理师、随访专员、康复治疗师
- 内容运营、质控人员、机构管理员、审计人员

## 1. 用户与合规基础

### 1.1 账号与身份
- 多方式登录：手机号验证码、第三方登录、匿名体验（可选）
- 实名认证（可选）：身份证/人脸（按政策与场景）
- 多成员家庭：家庭成员档案、代填权限
- 账号安全：设备管理、登录态管理、风控（异地登录/频繁尝试）

### 1.2 隐私与授权
- 隐私政策/用户同意：版本化、记录同意时间与渠道
- 分级授权：基础信息/健康数据/影像检验/随访数据/营销通知
- 撤回授权：功能降级、数据处理策略
- 数据导出/删除申请：申请→审核→执行→回执（留痕）

### 1.3 合规与审计（必备）
- 审计日志：关键业务全链路（测评、处方/建议、预警、转介、随访、发布、导出）
- 数据脱敏：不同角色可见字段差异
- 数据留存与归档策略：按法规/院内制度
- 安全：加密（传输/存储）、密钥管理、权限最小化

## 2. 健康档案与慢病管理（全量）

### 2.1 个人健康档案
- 基础信息：性别/年龄/身高体重/腰围/BMI
- 既往史/家族史/过敏史/手术史
- 生活方式：饮食、运动、睡眠、烟酒、职业压力
- 指标目标：按病种/人群模板 + 个性化调整

### 2.2 慢病专病档案
- 病种库（可扩展）：
  - 高血压、2型糖尿病、高脂血症、冠心病、心衰、房颤
  - 脂肪肝、肥胖、痛风、慢阻肺/哮喘、慢肾病
  - 颈腰椎/骨关节、卒中后、肿瘤康复（可选）
- 用药管理：药品/剂量/频次/开始结束、依从性记录、用药提醒
- 并发症与风险：肾病、视网膜病变、神经病变、心血管风险
- 复诊计划：门诊复诊、检验检查复查、用药调整记录

### 2.3 指标与数据采集
- 手动录入：血压、血糖（空腹/餐后）、体重、腰围、血脂（外院录入）、尿酸、峰流速等
- 设备接入（全量能力）：
  - 蓝牙血压计/血糖仪
  - 可穿戴：睡眠、心率、步数
  - 设备云平台对接（厂商API）
- 结构化数据校验：单位、范围、场景（晨起/睡前/餐后）

### 2.4 趋势与分析
- 7/30/90天趋势、统计（min/max/avg）、达标率
- 指标波动分析：触发因素标注（饮食、运动、情绪）
- 风险评估：分层（低/中/高），生成随访建议

## 3. 中医治未病能力（体质/辨证/调护）

### 3.1 体质辨识（九种体质）
- 体质问卷：题库、版本、计分规则、阈值配置
- 结果：主导体质 + 兼夹体质 + 解释 + 风险倾向
- 复评：30/90天提醒与对比
- 体质与慢病联动：模板选择、任务编排修饰

### 3.2 四诊信息采集（进阶）
- 问诊：症状库（口渴、乏力、胸闷、便秘、畏寒等）
- 望诊：舌象/面色图片采集、标准化描述（苔、质、色）
- 闻诊：汗味/口气等（可选）
- 切诊：脉象结构化（可选）
- **AI辅助（可选/高风险）**：舌象识别、症状结构化（需严格提示非诊断）

### 3.3 辨证分型与方案（可选/高风险）
- 辨证分型：按院内规范配置（证型库）
- 辨证依据：症状/舌脉/体质/指标
- 方案：以“调护+随访”为主，处方开方需走医疗合规与医生端流程

### 3.4 调护建议（全量、模板化）
- 建议结构：
  - 起居作息（睡眠、作息、工作节律）
  - 饮食调护（忌口/推荐、食疗方向、分人群提示）
  - 运动导引（八段锦、太极、散步、力量训练、禁忌）
  - 情志调护（呼吸放松、冥想、情绪记录）
  - 外治非药（泡脚、艾灸、穴位按揉、刮痧拔罐提示）
  - 风险提示（红旗症状/异常指标就医）
- 个性化：体质+慢病+指标趋势+反馈动态调整
- 版本管理：active/revised/archived，可对比差异

## 4. 随访管理（慢病核心闭环）

### 4.1 随访计划
- 计划模板：7/30/90天（按病种/体质/人群）
- 任务类型：
  - 指标上报（血压/血糖等）
  - 行为任务（运动、饮食、睡眠）
  - 调护任务（导引、外治、情志练习）
  - 用药提醒与依从性确认
  - 周期复盘（周报/月报）
- 计划版本化：调整留痕

### 4.2 打卡与依从性
- 今日任务清单：必做/选做
- 打卡记录：数值类+勾选类+备注
- 连续打卡、完成率、漏打提醒
- 依从性评分：按任务权重计算

### 4.3 随访记录与点评（专业端）
- 随访工作台：用户列表、风险排序、待处理提醒
- 随访记录：结构化模板 + 自由文本
- 专业点评：给出调整建议、下次随访时间
- 任务调整：增删任务、频次调整

### 4.4 随访报表
- 个体：周报/月报、达标率、趋势、建议变化
- 人群：按病种/科室/随访人员统计（活跃、达标、预警、转介）

## 5. 预警与分级处置（安全与运营）

### 5.1 预警规则（可配置）
- 指标异常：高危/中危/低危阈值（按病种）
- 依从性预警：连续未打卡/未上报
- 症状红旗：胸痛、呼吸困难、意识异常等（可配置）
- 复评到期提醒：体质/随访评估

### 5.2 预警事件管理
- 事件状态机：open → acked → closed（可选 escalated）
- 用户侧提示：复测/就医/联系随访
- 管理端处置台：确认、关闭、记录处置结论、转介建议
- 强制就医提示：高危必弹窗/必留痕

### 5.3 分诊与转介（可选）
- 转介意向：治未病门诊/慢病门诊/康复门诊
- 跟进状态：created → contacted → closed
- 与预约挂号系统对接（可选）

---

## 6. 院后康复与连续服务

### 6.1 院后康复计划
- 出院后7/14/30天康复模板（按手术/病种）
- 任务：训练、复诊提醒、用药提醒、风险症状监测
- 与随访共用打卡与预警

### 6.2 远程康复服务（可选）
- 视频/图文指导课程
- 康复师在线指导（IM/视频，需合规）
- 训练动作库与评估（可选）

---

## 7. 内容与课程（治未病科普体系）

### 7.1 内容库
- 文章/图文/视频/音频（可选）
- 分类与标签：体质、病种、饮食、运动、导引、情志
- 搜索、收藏、学习记录

### 7.2 内容运营后台
- 状态机：draft → pending_review → published → offline
- 审核流程：多级审核（可选）
- 敏感内容策略：医疗合规、夸大宣传拦截

---

## 8. 专业端（医生/管理师/康复师）

### 8.1 工作台
- 待随访列表、预警列表、风险排序
- 个体详情：档案+趋势+体质+计划+预警+历史建议

### 8.2 专业操作
- 计划调整、建议修订
- 处置记录、转介建议
- 随访结论与下次计划

### 8.3 质控（可选）
- 随访完成率、响应时效
- 处置闭环率

---

## 9. 管理后台（全量）

### 9.1 配置中心
- 病种与指标字典
- 目标值模板
- 体质问卷与计分规则
- 调护建议模板库
- 随访计划模板库
- 预警阈值与红旗症状
- 紧急资源（热线/急救提示文案）

### 9.2 运营与分析
- 漏斗：建档→计划启动→打卡→达标→复评→转介
- 统计：活跃、留存、达标率、预警率、处置时效
- 导出：按权限导出报表

### 9.3 权限与审计
- RBAC：角色、权限点、数据范围
- 审计查询与导出

## 10. 集成与接口（全量对标）

### 10.1 院内系统
- HIS/EMR：患者信息、诊断、就诊记录
- LIS：检验结果（血糖、血脂、尿酸等）
- 体检系统：体检报告导入与风险提示
- 预约挂号：转介与预约

### 10.2 设备与第三方
- 设备平台：血压计/血糖仪/可穿戴
- 消息平台：短信/微信模板消息/站内信
- 支付（可选）：服务包/会员

### 10.3 数据治理
- 主数据：用户、病种、指标、模板
- 数据质量校验与纠错

## 11. 状态机汇总（用于服务层与回归测试）

- ConstitutionAssessment: answering → submitted → scored → reported
- RecommendationPlan: active → revised → archived
- FollowupPlan: created → active → completed/terminated
- CheckIn: pending → done/missed
- AlertEvent: open → acked → closed (→ escalated 可选)
- ContentItem: draft → pending_review → published → offline
- ReferralRequest: created → contacted → closed

## 12. V1/V2/V3 拆包建议（便于落地）

### V1（闭环可演示）
- 高血压/糖尿病建档+指标上报+趋势
- 体质辨识（九体质）
- 调护建议（模板化）
- 30天随访计划+打卡
- 预警（指标异常/未打卡）+后台处置
- 内容库（文章）+审核发布
- 审计日志

### V2（增强运营与专业端）
- 专业端随访点评、计划调整、质控
- 更多病种、更多指标
- 复评体系、月报/人群报表
- 设备接入（部分）

### V3（招标全量对接）
- HIS/LIS/体检/预约挂号对接
- 多机构/多院区/多角色RBAC
- 支付与服务包、企业项目
- AI辅助舌象/症状结构化（谨慎）

---

## 13. 高风险合规提示
- 任何“处方/方药推荐”必须纳入医疗合规与医生端开立流程；V1建议只做调护建议与转介提示。
- 涉及设备数据、院内数据对接时，需落实授权、脱敏与审计。
