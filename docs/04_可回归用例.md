# 可回归用例（V1）

> 本文档覆盖 V1 闭环的完整回归测试矩阵，分为**单元测试**和**集成测试**两层。
> 测试框架：`pytest` + `pytest-asyncio`，数据库：SQLite in-memory（aiosqlite）。
> 运行命令：`pytest tests/ -v`

---

## 一、单元测试（`tests/services/`）

### 1.1 体质评分引擎 `test_constitution_scorer.py`

| # | 用例 | 验证点 |
|---|---|---|
| U-C-01 | 平和质高分 → 主导体质为 BALANCED | `main_type == BALANCED` |
| U-C-02 | 气虚质偏高 → 主导体质为 QI_DEFICIENCY | `main_type == QI_DEFICIENCY` |
| U-C-03 | 反向题（is_reverse=True）得分正确取反 | 答案5 → 计分等同于1 |
| U-C-04 | 兼夹体质：两种体质均倾向 → secondary_types 非空 | `len(secondary_types) >= 1` |
| U-C-05 | 转化分公式：原始分 → 百分制转换 | `converted_score ∈ [0, 100]` |
| U-C-06 | 全1分（最小答案）→ BALANCED 最高分 | 边界值正常处理 |
| U-C-07 | 全5分（最大答案）→ 偏颇体质最高分 | 边界值正常处理 |

### 1.2 预警引擎 `test_alert_engine.py`

## 二、集成测试（`tests/integration/`）

> 所有集成测试使用 `httpx.AsyncClient` 通过 FastAPI TestClient 调用完整 HTTP 栈。
> conftest 提供：in-memory SQLite DB、自动 `create_all` / 清表、测试客户端。

### 2.1 认证流程 `test_auth_flow.py`

| # | 用例 | 步骤 | 验证点 |
|---|---|---|---|
| I-AU-01 | 正常注册 | POST `/tools/auth/register` | `201`, `user_id` 存在 |
| I-AU-02 | 重复手机号注册 | 同上 | `400 VALIDATION_ERROR` |
| I-AU-03 | 正常登录 | POST `/tools/auth/login` | `200`, Cookie `access_token` 写入 |
| I-AU-04 | 密码错误登录 | POST `/tools/auth/login` | `401 VALIDATION_ERROR` |
| I-AU-05 | 未登录访问受保护接口 | GET `/tools/indicators` | `401` |
| I-AU-06 | 登出后 Cookie 清除 | POST `/tools/auth/logout` | `access_token` 被删除 |
| I-AU-07 | 同意隐私协议 | POST `/tools/auth/consent` | `200`, `consented: true` |
| I-AU-08 | 患者访问 ADMIN 专属接口 | GET `/tools/audit/` | `403 PERMISSION_ERROR` |

## 四、状态机边界测试矩阵

| 资源 | 合法流转 | 非法流转（期望 409） |
|---|---|---|
| ConstitutionAssessment | ANSWERING→REPORTED | REPORTED→ANSWERING |
| RecommendationPlan | ACTIVE→ARCHIVED | ARCHIVED→ACTIVE |
| FollowupPlan | CREATED→ACTIVE→COMPLETED | COMPLETED→ACTIVE |
| AlertEvent | OPEN→ACKED→CLOSED | CLOSED→OPEN |
| ContentItem | DRAFT→PENDING_REVIEW→PUBLISHED→OFFLINE | OFFLINE→PUBLISHED（直接）|

## 五、运行说明

```bash
# 全量回归
pytest tests/ -v

# 只跑单元测试
pytest tests/services/ -v

# 只跑集成测试
pytest tests/integration/ -v

# 只跑某一模块
pytest tests/integration/test_auth_flow.py -v

# 生成覆盖率报告（需安装 pytest-cov）
pytest tests/ --cov=app --cov-report=term-missing
```

> **注意**：集成测试使用 SQLite in-memory，与生产 PostgreSQL 行为略有差异（如 UUID 类型）。如遇 UUID 序列化问题，conftest 已通过 `String` 类型适配处理。
