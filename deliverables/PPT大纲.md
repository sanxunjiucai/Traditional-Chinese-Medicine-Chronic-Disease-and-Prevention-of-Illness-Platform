## 第一章：行业背景与痛点（第 3–4 页）

### 第 3 页：市场背景

- **数据**：我国慢性病患者超 4 亿，高血压、糖尿病是核心负担
- **政策**：中医治未病服务纳入基本公共卫生服务体系
- **趋势**：院后随访、慢病精细化管理成为三甲医院竞争焦点

> 可视化：漏斗图（健康人群 → 亚健康 → 慢病 → 并发症）

### 第 4 页：现有三大痛点

| 痛点 | 表现 | 影响 |
|---|---|---|
| 数据孤岛 | 体检、HIS、随访数据割裂，医生诊中无法一屏查看患者全貌 | 漏诊风险高 |
| AI 落不了地 | 大模型功能强大，但需切换独立界面，医生使用摩擦高 | AI 投资浪费 |
| 依从性低 | 方案下达即断联，患者打卡/随访/复诊脱节 | 慢病控制率低 |

## 第三章：核心亮点演示（第 8–16 页）

### 第 8 页：亮点一 — 诊中 AI 嵌入（最大差异化）

**Chrome 插件：治未病·诊中助手**

- **零感知嵌入**：安装后在任意 HIS/浏览器页面右侧悬浮绿竖条，点击展开
- **自动识别患者**：检测 URL 中的 `patient_id` 等参数，自动触发患者搜索 + AI 分析
- **三 Tab 侧边栏**：
  - Tab 1「当前风险」— 患者信息卡 + 风险等级徽章 + 风险证据溯源 + AI 摘要
  - Tab 2「历史方案」— 历史下达记录 + 状态机（ISSUED→IN_PROGRESS→FOLLOWED_UP→COMPLETED）
  - Tab 3「数据指标」— 机构累计分析次数 / 下达率 / 随访完成率
- **底部 AI 对话框** ⭐：自由文本提问，SSE 流式响应，支持图片/检验单上传
- **一键下达方案**：弹出确认框 → 勾选自动随访（3/7/14/30天）→ 推送患者 H5 通知

> 截图：插件侧边栏展开图 + 底部 AI 对话框

### 第 9 页：亮点二 — 管理端 Ctrl+K 智能助手

**全局 AI 工具链（49 个工具）**

- 任意管理端页面按 `Ctrl+K` 唤起，ESC 关闭
- 支持自然语言操作示例：
  - 「查找近期高风险患者」→ 调用患者搜索工具返回列表
  - 「帮我为张三创建7天后的随访计划」→ 调用 `create_followup_plan`
  - 「查看本月预警统计」→ 调用统计工具返回数据
  - 「跳转到预警处置台」→ `navigate_to` 工具自动跳转页面
- SSE 流式输出，实时展示 AI 思考过程与工具调用动画
- 最大 4 轮 Tool-Use 循环，AI 可自主链式调用多个工具

| 工具分类 | 数量 | 示例 |
|---|---|---|
| 患者档案 | 8 | 搜索、档案详情、指标、体质 |
| 方案管理 | 7 | 创建草稿、发布、历史版本对比 |
| 随访/召回 | 6 | 创建计划、任务列表、召回话术 |
| 工作台/统计 | 5 | 待处理事项、反馈摘要 |
| 模板知识库 | 3 | 指导/干预/宣教模板 |
| 系统导航 | 3 | 跳转页面、打开URL |
| 网络搜索 | 1 | 实时检索医学资讯 |

> 截图：Ctrl+K 对话框 + 工具调用动画

### 第 10 页：亮点三 — 三大模板知识库

**均已对接真实 API，支持场景筛选 + 一键复制**

| 模板库 | 场景芯片示例 | 功能 |
|---|---|---|
| 中医指导模板库 | 高血压、糖尿病、气虚质、失眠 | 筛选→预览→复制→创建新方案 |
| 干预方案模板库 | 针灸、推拿、高血压、减重 | 同上，涵盖针灸/推拿/中药等 |
| 宣教内容模板库 | 老年、儿童、体质科普、饮食 | 同上 |

- 医生一键调用模板生成新方案，大幅缩短书写时间
- 支持个人模板（PERSONAL）和机构公共模板（PUBLIC）分离

### 第 11 页：中医体质辨识

- 国标九种体质：平和质、气虚质、阳虚质、阴虚质、痰湿质、湿热质、血瘀质、气郁质、特禀质
- 72 道标准化问卷，5 级作答，自动评分与转化
- **断点续答**：`ANSWERING → SUBMITTED → REPORTED` 状态机
- 提交即自动生成个性化调护建议（体质+病种联合匹配）
- 管理端：体质评估列表 + 详情 + 出报告

> 截图：体质辨识问卷页 + 结果报告页

### 第 12 页：个性化调护建议 & 干预方案

- **调护建议五维**：起居作息 / 饮食调护 / 运动导引 / 情志调护 / 外治非药
- **干预方案 H5 打卡** ⭐：专属打卡页 `/h5/intervention/{id}`，患者手机端完成每日任务
- 版本归档：修订留痕，前后对比
- 管理端依从性统计：任务完成率/漏打率图表

> 截图：调护建议详情页 + 患者 H5 打卡页

### 第 13 页：健康指标管理 & 预警处置

**指标采集**
- 支持：血压、血糖（空腹/餐后）、体重、腰围
- 场景标注（晨起/睡前/餐后/随时）+ 备注
- 录入即触发预警引擎（零延迟）

**预警规则（数据库驱动，可配置）**

| 指标 | HIGH | MEDIUM | LOW |
|---|---|---|---|
| 血压（收缩压）| ≥180 mmHg | ≥160 | ≥140 |
| 血糖 | ≥11.1 mmol/L | ≥8.0 | ≥7.0 |

**处置流程**：`OPEN → ACKED → CLOSED`

> 截图：管理端预警处置台 + 患者 H5 指标录入页

### 第 14 页：随访管理 & 召回闭环

**随访管理**
- 按病种匹配 30 天模板，自动生成每日任务（指标/运动/用药/饮食/睡眠）
- 打卡状态机：`PENDING → DONE/MISSED`
- 依从性实时计算 + 逾期自动推送提醒

**召回闭环** ⭐
- AI 自动生成召回建议（含复诊理由和紧迫程度）
- 一键生成个性化电话召回话术（`get_recall_script` 工具）
- 医生接受/忽略 → 系统发送 `RECALL_VISIT` 通知给患者
- 形成「方案下达 → 随访 → 召回 → 复诊」完整商业闭环

> 截图：随访任务列表 + 召回建议页

## 第四章：技术架构与合规（第 17–19 页）

### 第 17 页：技术架构

```
Chrome 插件（诊中端）
    │ SSE 流式 + REST API
患者端 H5 / 管理端 PC（Jinja2 SSR）
    │ HTTPS / Cookie JWT
FastAPI 0.115（异步原生）
    │ 170+ 端点 · 25 工具模块
SQLAlchemy 2.0 async
    │
SQLite（演示）/ PostgreSQL（生产）
```

**AI 技术栈**

| 维度 | 管理端 Ctrl+K | 插件端 AI 对话 |
|---|---|---|
| 模型 | 智谱 GLM-4-air（OpenAI 兼容） | 同左（后端 SSE 代理） |
| 工具数 | 49 个（admin 模式） | 35 个（plugin 模式） |
| 输出 | SSE 流式 + 工具调用动画 | 同左，支持图片上传 |
| 迭代 | 最大 4 轮 Tool-Use 循环 | 最大 4 轮 |

### 第 18 页：安全合规设计

| 方面 | 实现 |
|---|---|
| 传输安全 | HTTPS + Cookie SameSite=Lax + HttpOnly（防 XSS） |
| 存储安全 | 密码 bcrypt，JWT 密钥从环境变量注入 |
| 权限最小化 | RBAC 三级角色，接口级最小权限校验 |
| 数据审计 | 16 类关键 action 全量写 AuditLog（IP + 操作前后值） |
| 隐私授权 | ConsentRecord 版本化记录，含时间戳与渠道 |
| 数据隔离 | 越权访问返回 404，不暴露存在性 |
| AI 合规 | 所有 AI 输出标注「仅供参考，不构成诊断」 |

### 第 19 页：合规边界声明

- V1 **不涉及处方**：调护建议属非医疗健康指导
- V1 **不涉及设备对接**：手动录入，无第三方数据授权风险
- AI 辅助（V3）：舌象识别严格标注「仅供参考，不构成诊断」
- 数据留存：健康数据 ≥5 年，审计日志 ≥3 年
