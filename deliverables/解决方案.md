# 中医慢病与治未病平台 — 解决方案

> 版本：V2.0 · 2026-03-08
> 适用场景：三甲医院治未病科/慢病管理中心信息化招标、院内自研落地

---

## 一、背景与目标

### 1.1 行业背景

- 我国慢性病患者超 4 亿，高血压、糖尿病是核心负担；中医治未病体系被纳入基本公共卫生服务。
- 现有痛点：**数据孤岛**（体检、HIS、随访数据割裂）、**AI 落地摩擦高**（大模型需切换界面，医生不愿用）、**依从性低**（打卡与随访脱节，方案发完即断联）。

### 1.2 建设目标

| 目标维度 | 具体指标 |
|---|---|
| 三端协同 | Chrome 插件（诊中端）· 管理端 PC · 患者端 H5 无缝联动 |
| AI 落地 | 医生无需切换界面，HIS 诊中嵌入 AI 侧边栏；管理端 Ctrl+K 自然语言操作 |
| 闭环覆盖 | 建档 → 体质辨识 → AI 风险分析 → 一键下达方案 → 随访打卡 → 召回闭环 全链路贯通 |
| 数据合规 | 操作全链路审计日志、隐私授权版本化、RBAC 权限最小化 |
| 可扩展 | 病种/指标/模板/AI 工具均为字典/配置驱动，支持 V2/V3 扩容 |
| 可演示 | DEMO_MODE 自动 Seed 71 名患者数据，10 分钟内可本地启动展示 |

## 三、核心功能模块

### 3.1 诊中嵌入端（Chrome 插件）

这是平台与传统方案最大的差异化。

**自动化流程**：
1. 医生在 HIS 系统打开患者页面（URL 含 `patient_id` 等参数）
2. 插件自动识别患者 ID，向后端发起患者搜索 + AI 风险分析
3. 右侧侧边栏自动展开，展示三 Tab 内容

**侧边栏三 Tab**：
- **Tab 1 当前风险**：患者信息卡 + 风险等级（高/中/低）+ 风险证据溯源 + AI 摘要 + 一键下达方案
- **Tab 2 历史方案**：历史下达记录 + 状态机流转（ISSUED → IN_PROGRESS → FOLLOWED_UP → COMPLETED）
- **Tab 3 数据指标**：机构累计分析次数 / 下达率 / 随访完成率

**底部 AI 对话框**：
- 医生可在侧边栏底部直接提问（如「这个患者适合哪些中药？」）
- 后端 SSE 流式响应，支持图片/检验单上传（OCR 识别）
- 调用 35 个 plugin 模式工具，最大 4 轮 Tool-Use 循环

**一键下达方案**：
- 弹出确认框，显示患者信息 + 方案内容预览
- 勾选「自动创建随访提醒」+ 选择时间（3/7/14/30天）
- 确认后推送 `PLAN_ISSUED` 通知到患者 H5

### 3.2 管理端 AI 智能助手（Ctrl+K）

任意管理端页面按 `Ctrl+K` 唤起，支持自然语言操作 49 个工具：

| 工具分类 | 工具数 | 典型操作 |
|---|---|---|
| 患者档案管理 | 8 | 搜索患者、查档案、查指标、查体质 |
| 方案管理 | 7 | 创建草稿、发布方案、查历史版本、对比方案 |
| 随访/召回 | 6 | 创建随访计划、查任务、生成召回话术 |
| 工作台/统计 | 5 | 待处理事项、患者反馈摘要 |
| 模板知识库 | 3 | 查指导/干预/宣教模板 |
| 系统导航 | 3 | 跳转页面、打开 URL |
| 网络搜索 | 1 | 实时检索医学资讯、临床指南 |

SSE 流式输出 + 工具调用动画，最大 4 轮 Tool-Use 循环。

### 3.3 用户与合规基础

- **认证**：手机号/邮箱 + 密码登录，HttpOnly Cookie JWT，`access_token_expire` 可配置
- **隐私授权**：`ConsentRecord` 版本化记录，含同意时间与渠道
- **RBAC**：`PATIENT` / `PROFESSIONAL` / `ADMIN` 三级角色，接口级权限拦截
- **审计日志**：16 类关键 action 自动写入，支持时间范围 + 用户 + 动作筛选

### 3.4 健康档案与慢病管理

- **多类型档案**：普通 / 儿童 / 女性 / 老年 / 重点人群 / 家庭 / 活动档案，共 71 名演示患者
- **个人健康档案**：基础信息、生活方式（吸烟/饮酒/运动/睡眠/压力）、Upsert 语义
- **慢病专病档案**：可扩展病种字典（V1 支持高血压、2型糖尿病）、用药记录、目标值
- **指标采集**：血压、血糖、体重、腰围；支持场景标注（晨起/餐后）
- **指标联动预警**：录入即触发预警引擎，无额外调用

### 3.5 中医治未病（体质辨识）

- **九种体质问卷**：72 道题、版本化题库、反向题处理
- **评分算法**：原始分 → 百分制转化分，主导体质 + 兼夹体质
- **状态机**：`ANSWERING → SUBMITTED → REPORTED`，断点续答
- **联动调护建议**：提交即自动生成个性化 RecommendationPlan

### 3.6 三大模板知识库

均已对接真实 API，支持场景芯片筛选、排序、一键复制：

| 模板库 | API 路径 | 复制接口 |
|---|---|---|
| 中医指导模板库 | `GET /tools/guidance/templates` | `POST /tools/guidance/templates/{id}/copy` |
| 干预方案模板库 | `GET /tools/intervention/templates` | `POST /tools/intervention/templates/{id}/copy` |
| 宣教内容模板库 | `GET /tools/education/templates` | `POST /tools/education/templates/{id}/copy` |

### 3.7 随访管理与干预打卡

- **随访计划**：按病种匹配 30 天模板，自动生成每日任务（指标/运动/用药/饮食/睡眠）
- **打卡系统**：CheckIn 状态机（`PENDING → DONE/MISSED`）
- **患者 H5 干预打卡页** ⭐：`/h5/intervention/{id}`，专属打卡界面
- **依从性计算**：`完成任务数 / 总应完成任务数`，实时图表

### 3.8 预警与处置

- **预警规则**（数据库驱动可配置）：HIGH/MEDIUM/LOW 三级
  - 血压：收缩压 ≥180（HIGH）/ ≥160（MEDIUM）/ ≥140（LOW）
  - 血糖：≥11.1（HIGH）/ ≥8.0（MEDIUM）/ ≥7.0（LOW）
- **事件状态机**：`OPEN → ACKED → CLOSED`
- **管理端处置台**：按 severity/status 筛选，记录 handler_note

### 3.9 召回建议（RECALL_VISIT 闭环）⭐

- AI 自动分析患者随访情况，生成召回建议（含复诊理由和紧迫程度）
- 支持 AI 生成个性化电话召回话术（`get_recall_script` 工具）
- 医生接受/忽略召回建议 → 系统发送 `RECALL_VISIT` 通知给患者
- 形成「方案下达 → 随访打卡 → 依从性分析 → 召回建议 → 复诊转化」完整商业闭环

### 3.10 在线咨询与通知推送

- **在线咨询**：患者 H5 发起工单 → 医生工作台回复 → 气泡对话视图
- **五类通知**：`🚨 风险预警 / 📋 方案下达 / 📅 随访提醒 / 💬 咨询回复 / 🔔 召回建议`
- 医生回复后自动推送通知到患者 H5

### 3.11 内容库（治未病科普）

- **状态机**：`DRAFT → PENDING_REVIEW → PUBLISHED → OFFLINE`
- **分角色视图**：患者只见 PUBLISHED；管理员全量管理
- **标签体系**：支持体质/病种/饮食/运动等多标签过滤

## 四、技术选型

| 层 | 技术 | 理由 |
|---|---|---|
| Web 框架 | FastAPI 0.115 | 异步原生、自动 OpenAPI、高性能 |
| ORM | SQLAlchemy 2.0 asyncio | 异步 session，类型安全 |
| 数据库 | PostgreSQL（生产）/ SQLite（演示） | 生产高并发，演示零依赖 |
| 模板渲染 | Jinja2 SSR | 无需前端构建，快速可演示 |
| 认证 | JWT + HttpOnly Cookie | CSRF 防护，无 XSS 泄漏风险 |
| 密码加密 | bcrypt（passlib） | 行业标准，抗彩虹表 |
| AI 模型 | 智谱 GLM-4-air（OpenAI 兼容格式） | 国产高性价比，可配置替换 |
| AI 接入点 | `https://open.bigmodel.cn/api/paas/v4` | 可通过 `.env` 配置替换 |
| 流式输出 | SSE（Server-Sent Events） | 浏览器原生支持，无需 WebSocket |
| 浏览器插件 | Chrome Manifest V3 | 现代扩展规范，Edge/Brave 兼容 |
| 前端 | 原生 HTML/JS + Tailwind CDN | 零构建，快速迭代 |
| 迁移 | Alembic | 版本化 DDL，生产安全 |

## 六、安全合规设计

| 方面 | 实现 |
|---|---|
| 传输安全 | HTTPS（生产），Cookie SameSite=Lax + HttpOnly |
| 存储安全 | 密码 bcrypt，JWT HS256 密钥从环境变量注入 |
| 权限最小化 | 每个接口声明最低角色要求，deps 层统一校验 |
| 数据审计 | 关键操作全量写 AuditLog（action + old/new values + IP） |
| 隐私授权 | ConsentRecord 版本化，含时间戳与渠道 |
| 数据隔离 | 患者只能访问自己数据，越权返回 404（不暴露存在性） |
| AI 合规 | 所有 AI 输出标注「仅供参考，不构成诊断」 |
| 密钥安全 | AI API Key 仅从 `.env` 读取，不随系统环境变量传播 |

## 十、风险与合规提示

1. **处方禁止**：V1 仅输出「调护建议」，不涉及处方、方药推荐；如需处方须接入医生端开立流程并满足互联网诊疗规范。
2. **设备数据**：接入第三方设备平台需落实用户授权与数据脱敏。
3. **AI 功能**：所有 AI 辅助输出须标注「仅供参考，不构成诊断」，建议灰度验证后上线。
4. **数据留存**：按《网络安全法》《个人信息保护法》，健康数据建议留存 ≥5 年，审计日志 ≥3 年。
5. **模型合规**：当前接入智谱 GLM-4-air，如需替换模型（如 Claude、GPT-4），仅修改 `.env` 三行配置即可，无需改代码。
