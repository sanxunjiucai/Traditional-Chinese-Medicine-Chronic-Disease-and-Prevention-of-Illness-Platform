## 三、Chrome 插件安装与配置

> 这是系统与传统方案最大的差异化：**医生无需切换界面**。

### 3.1 安装插件

1. Chrome 访问 `chrome://extensions/`
2. 右上角开启「开发者模式」
3. 点击「加载已解压的扩展程序」→ 选择项目目录下的 `chrome_extension/` 文件夹
4. 工具栏出现绿色 🌿 图标，安装完成
5. 支持 Chrome 93+、Edge、Brave 等 Chromium 浏览器

### 3.2 首次配置

1. 点击工具栏插件图标，弹出配置页
2. **后端服务地址**：确认为 `http://localhost:8015`（插件会自动迁移旧端口）
3. **AI 配置**（可选，供插件端底部 AI 对话使用）：
   - Anthropic/GLM API Key：填入你的密钥
   - Base URL：`https://open.bigmodel.cn/api/paas/v4`（GLM）或留空（用 Claude 官方）
   - Model：`glm-4-air` 或其他
4. 点击「保存配置」，连接成功后提示绿色 ✓

> 实际演示中 AI 对话走后端 SSE 代理，无需在插件填入 Key，后端 `.env` 配置生效即可。

### 3.3 使用前登录

浏览器访问 `http://localhost:8015/login` 用 `admin@tcm` 登录（插件通过 Cookie 认证）。

### 3.4 演示方法

**方法一：URL 参数自动触发（推荐）**

在地址栏访问任意带 `patient_id` 参数的页面：
```
http://localhost:8015/gui/admin/archives?patient_id=张
```
→ 插件检测到参数后自动向后台发起患者搜索 + AI 风险分析，右侧侧边栏展开显示结果

**方法二：弹窗手动搜索**

1. 打开任意页面（如 `http://localhost:8015/gui/admin/archives`）
2. 点击工具栏插件图标 → 在「手动搜索患者」框输入患者姓名（如"李"）
3. 点击「搜索」→ 侧边栏自动展开并显示分析结果

### 3.5 侧边栏三 Tab 详解

插件侧边栏固定在页面右侧，点击 ◀ 可收起为绿色竖条，点击竖条再展开。

#### Tab 1：当前风险

| 区域 | 内容说明 |
|---|---|
| 患者信息卡 | 姓名 + 档案编号 + 风险等级徽章（高🔴/中🟠/低🟢） |
| 风险证据 | 每条证据可点击展开溯源（检测值/参考范围/来源/日期） |
| AI 摘要 | 风险分析自然语言摘要 |
| 下达按钮 | 「⚡ 确认下达方案」→ 弹出确认框 |
| 跳转按钮 | 「完整分析 ↗」→ 新标签页打开平台完整分析页 |

**下达方案确认框**：
- 显示患者姓名 + 方案内容预览
- 勾选「自动创建随访提醒」（默认开启）+ 选择随访时间（3/7/14/30天）
- 确认后创建指导记录并推送患者通知；若勾选随访，自动创建随访任务

#### Tab 2：历史方案

展示该患者所有历史下达记录，支持**方案状态流转**：

| 当前状态 | 可操作 |
|---|---|
| 已下达（ISSUED） | → 标记进行中 |
| 进行中（IN_PROGRESS） | → 标记已随访 / 调整方案 |
| 已随访（FOLLOWED_UP） | → 标记已复评 / 完结 |
| 已复评（RE_ASSESSED） | → 完结 |
| 已完结（COMPLETED） | — |

#### Tab 3：数据指标

| 指标 | 说明 |
|---|---|
| 总分析次数 | 本机构累计 AI 分析触发次数 |
| 已下达方案 | 通过插件下达的方案总数 |
| 方案下达率 | 中高风险患者中已下达方案的比例（进度条） |
| 随访完成率 | 已创建随访任务中已完成比例（进度条） |

### 3.6 底部 AI 对话框 ⭐ 新功能

侧边栏底部有独立 AI 对话区域（类似 ChatGPT 界面）：
- 支持自由文本提问（如「这个患者适合哪些中药？」「帮我查询最新的高血压指南」）
- SSE 流式响应，实时展示 AI 思考过程和工具调用动画
- 支持图片/检验单上传（OCR 识别后分析）
- 调用 **35 个 plugin 模式工具**：患者档案/方案/随访/召回/模板/网络搜索等
- 最大 4 轮 Tool-Use 自主循环

## 四、核心演示：三条商业价值链（约 15 分钟）

> 这是系统最核心的竞争力，建议从这里开始演示。

### 🔗 价值链 1：诊中 AI 识别 → 一键下达方案

**场景**：医生正在接诊，插件自动扫描患者数据并展示风险分析，一键下达方案。

#### 方式A：Chrome 插件演示（最具冲击力）

1. 确认已登录平台（`admin@tcm`）
2. 在地址栏访问：`http://localhost:8015/gui/admin/archives?patient_id=张`
3. 插件自动展开侧边栏，显示「张」姓患者的风险分析
4. 查看 Tab 1：风险等级徽章 + 证据溯源 + AI 摘要
5. 点击「⚡ 确认下达方案」→ 勾选"自动创建7天随访提醒"→ 确认
6. Toast 提示：方案已下达，通知已推送 ✓

**底部 AI 对话演示**：
- 在侧边栏底部输入「这个患者适合推拿吗？」→ AI 实时流式回答
- 输入「帮我查最新高血压中医诊疗指南」→ AI 调用网络搜索工具

#### 方式B：综合就诊台（管理端）

> 路径：`/gui/admin/visit-station`（侧边栏 → 中医管理 → 综合就诊台）

1. 搜索患者：左列搜索框输入姓名（如"王"），点击患者卡片选中
2. 查看临床数据：中列自动加载该患者的检验/就诊/处方历史
3. 触发 AI 分析：点击「开始 AI 分析」
4. 查看风险结果：风险等级徽章 + 风险因素 + AI 综合摘要
5. 生成调理方案：点击「生成中医方案」→ AI 输出方案（含饮食/运动/中药/穴位）
6. 一键下达：填写方案标题 → 点击「下达给患者」

---

### 🔗 价值链 2：患者收到推送 → 查看方案 → H5 打卡

**场景**：患者回家后收到方案通知，查看并开始干预打卡。

> 切换账号：退出 → `patient@tcm` 登录 → `/h5/home`

1. **首页铃铛**：欢迎卡片右上角出现 🔔 红点（未读通知数）
2. **通知中心** `/h5/notifications`：
   - 展示通知：🚨风险预警 / 📋方案下达 / 📅随访提醒 / 💬咨询回复 / 🔔召回建议
   - 点击「方案下达」通知 → 跳转调理方案详情页
3. **方案详情** `/h5/plan/{id}`：
   - AI 生成的调理方案 Markdown 渲染展示
   - 底部「就方案咨询医生」按钮 → 进入咨询发起页
4. **干预打卡** `/h5/intervention/{id}`：
   - 每日任务清单（指标上报/运动/用药/饮食/睡眠）
   - 打卡后实时更新依从性进度条

---

### 🔗 价值链 3：患者在线咨询 → 医生回复 → 复诊召回

**患者侧**（`patient@tcm`）：

1. 首页快捷入口「在线咨询 💬」→ `/h5/consultation/new`
2. 填写标题 + 问题描述 + 选择普通/紧急 → 提交
3. 跳转咨询详情页 → 可继续追问

**医生侧**（切换回 `admin@tcm`）：

1. 工作台底部「待回复咨询」列表 → 点击进入
2. 咨询详情：气泡对话视图（患者消息左灰/医生消息右绿）
3. 输入回复：「建议来院开方，下周二我有门诊，可以直接挂号就诊」
4. 回复后：系统自动推送「💬 咨询回复」通知到患者 H5

**召回闭环演示**：

1. 管理端 → 中医管理 → AI风险诊断 → 召回建议 Tab
2. 查看 AI 生成的召回建议（含复诊理由和紧迫程度）
3. 点击「生成召回话术」→ AI 生成个性化电话脚本
4. 点击「接受召回」→ 系统发送 `RECALL_VISIT` 通知给患者
5. 患者 H5 收到「🔔 建议复诊」通知 → 点击联系医生

```

Step 7  patient@tcm → 在线咨询 → 发起新咨询
        → 填写「血糖控制不好，方案中的中药需要医院开吗？」→ 提交

Step 8  切换回 admin@tcm → 工作台 → 待回复咨询
        → 点击进入咨询详情 → 回复
        → 查看患者端通知中心：收到「💬 咨询回复」

── 第四幕：Ctrl+K 智能助手 ─────────────────────────────── (~2分钟)

Step 9  admin@tcm 任意管理端页面 → 按 Ctrl+K
        → 输入「查看今日高风险患者」→ AI 实时返回列表
        → 输入「跳转到预警台」→ AI 调用 navigate_to 工具自动跳转

── 第五幕：商业数据看板 ─────────────────────────────────── (~3分钟)

Step 10 统计分析 → 业务运营统计
        → 展示：复诊转化率 / 咨询收入估算 / 月度趋势图
        → 商业价值估算卡片

Step 11 (可选) 召回建议演示
        → AI 风险诊断 → 召回建议 Tab
        → 点击「生成召回话术」→ AI 生成个性化电话脚本
        → 接受召回 → 患者收到 RECALL_VISIT 通知

## 十一、AI 功能配置

### .env 配置

```env
# AI 模型（智谱 GLM-4-air，OpenAI 兼容）
ANTHROPIC_API_KEY=你的API密钥
ANTHROPIC_BASE_URL=https://open.bigmodel.cn/api/paas/v4
ANTHROPIC_MODEL=glm-4-air

# 数据库
DATABASE_URL=sqlite+aiosqlite:///./demo.db
DEMO_MODE=true
```

> 系统直接解析 `.env` 文件读取 AI 配置，不受系统环境变量覆盖影响。

### AI 功能对应关系

| 功能 | 模式 | 工具数 | 触发方式 |
|---|---|---|---|
| 管理端 Ctrl+K | admin | 49 个 | 任意管理端页面按 Ctrl+K |
| 插件端底部对话 | plugin | 35 个 | 插件侧边栏底部输入框 |
| 插件端自动分析 | — | — | 检测到患者ID自动触发 |

### 无 API Key（规则引擎兜底）

- 风险分析：基于阈值规则（血糖≥7.0 / 血压≥140 等）
- 方案生成：返回体质对应的默认模板方案
- Ctrl+K 和插件底部 AI 对话返回「未配置 API Key」提示
- 所有其他功能正常可用
